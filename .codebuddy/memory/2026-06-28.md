## P1 基础类型层实现完成（C1–C2）

实现 error / ast / value 三个包，20 个测试全绿，moon check/test/fmt/info 全通过。

### 关键实现细节
- `error` 包：`JsonataError` suberror（5 变体：Syntax/Runtime/Type/Signature/Guardrail），derive(Debug, Eq)，便捷构造函数。
- `ast` 包：`Ast` enum（15 变体）+ `Step`/`StepKind`/`StepStage`/`SortTerm`，derive(Debug, Eq)。路径统一为 `Path(Array[Step])`，移除了与 StepKind 冗余的顶层 Name/Index 等变体（已更新 architecture.md）。
- `value` 包：`JsonataValue`（pub(all) enum，四态）+ `JsonataFunc`（手动 impl Debug，闭包仅显示元数）+ `EvalContext`（护栏：depth/steps，默认 1000/1_000_000）。序列展平/单例提升/Undefined 传播。

### MoonBit 语法踩坑记录
- `moon.pkg` 的 import 用花括号 `{}`，非圆括号。
- enum 构造器位置字段不能写 `name : Type`，需用 `Type`（位置）或 `name~ : Type`（标签）。
- struct 字段不能用 `~`（`~` 仅用于函数标签参数）；`pub struct` 字段默认公开，无需 `pub`。
- 手动 impl trait 语法：`impl Debug for T with method(self) { ... }`，非 `pub impl ... { pub fn ... }`。
- `derive(Show)` 已废弃，用 `derive(Debug)`；`assert_eq` 需类型实现 Show，对自定义类型用 `assert_true(a == b)` + derive(Eq)。
- `not()` 已废弃，用 `!expr`；`Array` 无 `extend` 方法，用循环 push。
- `test "..." raise {` 语法错误——测试块默认可抛错，无需 `raise` 标注。
- suberror 变体有标签字段时，模式匹配用 `Variant(..)` 或 `Variant(label=value)`，非 `Variant(_)`。
- 跨包类型需加包前缀（如 `@error.JsonataError`）。
- 闭包类型语法：`(A, B) -> R raise E`，非 `(A, B) -> R!E`。

## P2 词法分析 + P3 语法分析完成（C3–C5）

lexer 包（C3）与 parser 包（C4–C5）实现完成，49 个测试全绿，moon check/test/fmt/info 全通过。

### lexer 包
- `TokenKind` enum（40+ 变体）：字面量/标识符/变量/运算符/括号/路径操作符。
- `Token` struct：kind + pos（字符偏移）。
- `lex(String) -> Array[Token]`：递归扫描，支持字符串转义（\n\t\r\"\'\\/b f \uXXXX）、数字（整数/小数/指数）、注释（单行// 块/* */）、变量（$name/$$/$）。
- 关键修正：`.*` 与 `.**` 区分——`.*` 后非 `*` 产生 DotStar 单 token；`.*` 后跟 `*` 产生 Dot + DoubleStar 两 token。

### parser 包
- `parse(Array[Token]) -> Ast`：递归下降，表达式优先级链：
  block → chain(~>) → bind(:=) → sort(^) → or(||) → and(&&) → comparison(= != < <= > >=) → concat(&) → additive(+ -) → multiplicative(* / %) → unary(- !) → postfix(. .* ** [] () ) → primary
- 支持：字面量/路径/下标/切片/谓词过滤/通配符/递归降序/序列构造/分组表达式/Lambda/函数调用/函数链/排序/绑定/块表达式/变量($ $name $$ @)。
- 关键修正：
  - `function` 关键字检查需在 Ident case 之前（否则被当作路径）。
  - end-of-input 哨兵 Null 需检测 at_end 避免误解析为 null 字面量。
  - 绑定表达式 `:=` 后需消费 `;` 分隔符再解析 body。
  - `advance()` 返回 Token 需 ignore；`unreachable()` 不存在用 `abort(String)`。

### evaluator 包
- `eval(Ast, EvalContext) -> JsonataValue`：模式匹配 AST 节点求值。
- 路径求值：`eval_path` 逐步遍历，支持字段访问/通配符/下标/切片/谓词过滤/递归降序。
- Var 特殊处理：`$[0]` 解析为 `Path([Filter(Var("")), Index(0)])`，求值时第一步 Filter(Var(name)) 用 Var 作为基值。
- 二元运算：算术(+ - * / %)/比较(= != < <= > >=)/逻辑(&& ||)/字符串拼接(&)，Undefined 传播。
- 一元运算：`-` `!`。
- 序列构造/绑定/块/Lambda/函数调用/函数链/分组表达式/排序。
- 关键修正：
  - `?` 错误传播已废弃，raise 函数中直接调用即可。
  - `Json::True`/`False`/`Null` 是只读变体，构造用 `Json::boolean()`/`Json::null()`，模式匹配用 `Json::True`/`Json::False`。
  - 结构体函数字段调用：`(f.invoke)(args, ctx)` 而非 `f.invoke(args, ctx)`。
  - Lambda 闭包需显式类型注解（参数+返回+raise）。
  - `sort_by` 闭包不能 raise，需预计算排序键。
  - for 循环元组解构用 `for kv in pairs { let (a, b) = kv; ... }`。

## 实现缺口审计（对照 docs/）

对照 architecture.md / design-decisions.md 审计，主要缺口分三类：

### A. 语义正确性缺口（影响 JSONata 兼容性）
1. **`@`/`$$` 上下文未正确实现**：`eval_var` 中 `@`（当前项）和 `$$`（父级）都简化为 `ctx.root()`。JSONata 中 `@` 应为谓词过滤时的当前项，`$$` 应为父上下文。需在 EvalContext 增加 `current`/`parent` 字段。
2. **谓词过滤 `access_filter` 语义错误**：对每个 item 求值时未将 item 绑定为 `@`/当前上下文，导致 `$x.age > 25` 这类过滤无法访问当前 item。当前实现 `ctx.with_depth(() => eval(expr, ctx))` 对所有 item 求值相同表达式，结果一致。
3. **`&` 合并运算符未实现**：文档要求 `&` 既是字符串拼接也是对象合并，当前 `eval_binary` 的 `"&"` 只调用 `concat_str`，未处理对象合并。
4. **部分应用未实现**：文档 P7 要求 Lambda 部分应用，当前 `eval_apply` 仅支持完整参数调用。
5. **函数签名系统未实现**：文档要求"类型化参数与返回值、类型检查"，当前仅有元数检查，无类型签名。

### B. 函数缺口（已实现 47 个，文档要求 60+）
已注册函数清单（47个）：string/number/boolean/type/eval/exists/not/now/millis/base64encode/error/keys/values/merge/spread/lookup/each/sift/map/filter/reduce/count/append/flatten/sort/reverse/join/sum/max/min/abs/floor/ceiling/round/power/random/substring/length/trim/split/concat/uppercase/lowercase/contains

缺失的常用函数：
- 字符串：$replace/$substringBefore/$substringAfter/$pad/$formatNumber
- 数值：$sqrt/$pi/$e/$formatBase
- 数组：$distinct/$single/$partition/$zip
- 日期：$fromMillis/$toMillis/$formatDateTime（$now/$millis 为占位）
- 正则：$match/$matches（R1 风险，MoonBit 无正则库）
- 编码：$base64decode/$encodeUrlComponent/$decodeUrlComponent
- 其他：$assert/$evalExpression/$shallow/$deep

### C. 工程缺口
1. **CLI 未支持命令行参数**：wasm-gc 目标限制，`cmd/main` 仅为演示模式，无法 `moonata '<expr>' --data file.json`。需 native 目标 + `@argv` 或 `@stdio`。
2. **JSONata 官方测试套件未移植**：文档 C17 要求移植官方测试套件核心用例，当前仅自编写测试。
3. **`@` 当前项在 EvalContext 无字段**：`EvalContext` 仅有 `root`，缺 `current`/`parent` 字段支持 `@`/`$$` 语义。
4. **`$random` 为占位**：返回固定 0.5，非真随机。
5. **`$now`/`$millis` 为占位**：返回固定值，非真实时间。
6. **`$base64encode` 为占位**：直接返回原字符串，未编码。
